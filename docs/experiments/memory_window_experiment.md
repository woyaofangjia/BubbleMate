# 记忆窗口调优实验记录

> **数据来源**：`scripts/real_experiment_runner.py` 真实运行结果  
> **实验时间**：2026-06-25

## 实验背景

面试核心问题：**"记忆窗口保留几轮？为什么是5轮不是10轮？"**

## 实验设计

测试对话（6轮）：
1. 用户：给我推荐一款好喝的
2. Agent：推荐芝芝莓莓、杨枝甘露...
3. 用户：杨枝甘露多少钱？
4. Agent：18元...
5. 用户：太甜了，下次我要少糖的
6. Agent：已记录您偏好少糖...

关键测试：上下文是否包含"杨枝甘露"和"少糖"。

## 真实实验结果

| 窗口大小 | 消息数 | 有摘要 | 记住饮品 | 记住偏好 | 响应时间 |
|---------|--------|--------|---------|---------|---------|
| **3轮** | 4 | ✓ | ✓ | ✓ | 0.2ms |
| **5轮** | 6 | ✓ | ✓ | ✓ | 0.0ms |
| **10轮** | 6 | ✗ | ✓ | ✓ | 0.0ms |

## 实验分析

### 为什么选5轮？

**3轮的问题**：
- 消息数只有4条（压缩后），虽然能记住关键信息，但对话历史被过度压缩
- 对于实际奶茶店客服场景（点单→确认→付款→配送→售后），3轮窗口可能丢失中间状态

**5轮的优势**：
- 消息数6条，保留了完整的对话上下文
- 触发了摘要压缩（`has_summary: true`），历史信息被有效提炼
- 响应时间极快（0.0ms）

**10轮的问题**：
- 消息数也是6条，说明在6轮对话中10轮窗口没有被充分利用
- 没有触发摘要压缩（`has_summary: false`），意味着更长的对话才会触发
- 对于奶茶店场景，10轮对话过于冗长，实际很少出现

### 实体提取的效果

无论窗口大小，系统都能记住"杨枝甘露"和"少糖"，原因是：

```python
# 从对话中自动提取的关键实体
preferences = {
    "sugar_level": "少糖",
    "favorite_drinks": ["杨枝甘露"]
}
```

这些实体**显式注入**到上下文中，不受窗口大小限制。
