## 核心能力验证

### 1. 参数缺失 → 自动反问

当用户未提供必填参数时，系统不会静默失败或报错，而是：

```
我需要更多信息才能帮您查询：
  - order_id: 订单编号

请提供这些信息。
```

**与普通调API的区别**：普通API返回400错误码，需要前端自己处理；Agent层自动转换为友好提示。

### 2. 业务错误 → 引导用户

当订单不存在时，系统不是简单说"404"，而是：

```
抱歉，订单不存在。请检查订单号是否正确，或者您可以尝试：
  - 换个关键词搜索
  - 联系人工客服核实
```

### 3. 三层降级链路

```
用户请求
    ↓
[第一层] 参数验证 → 缺失则反问
    ↓
[第二层] 工具调用 → 失败则重试
    ↓
[第三层] 结果处理 → 空结果则引导换关键词
    ↓
生成回复
```
